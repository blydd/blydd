idea用maven创建springmvc项目

1、DEA创建项目

新建一个maven project，并且选择webapp原型。

勾选:Create from archetype

选择:org-apache.maven.archetype:maven-archetype-webapp

然后点击next

输入GroupId和ArtifactID,ArtifactID最好和你的项目一名一样,然后next

为了快一点创建，我们添加一个属性值，如图中亮的所示，点右边的加号，name=archetypeCatalog value=internal。

点击next,接下来idea开始创建项目

然后等待maven创建项目成功

这样我们的项目初始的框架就弄好了。

2、添加pom依赖

创建好之后的项目目录如图所示

我们打开其中的pom.xml，添加我们的依赖。

3、添加框架支持

配置完pom.xml之后，我们在idea中要添加一下框架的支持。

右击我们的项目文件夹，选择add framework support (如果不显示spring,解决见文末)

然后在窗口中分别选中spring和springmvc，并且选择spring时，记得勾选springconfig.xml

点击ok之后，我们会发现WEB-INF文件夹下多出了两个文件:applicationContext.xml dispatcher-servlet.xml

这个就是我们之后要配置的文件，先不用管。

**4、完善目录结构

**首先我们在src/main文件夹下创建java和resources文件夹.

右击java文件夹:Mark Directory as –> Sources Root

右击resources文件夹:Mark Directory as –> Resources Root

这样我们配置前的工作就完成了，接下来就是对springmvc进行配置。我把两种配置的方法分成两部分，以供参考。

IDEA的项目右键选择Add Frameworks Support中发现没有如下图的Spring

这个的原因是，在Project Structure中已经有了Spring的Modules，所以无法再创建新的

![image-20200629171656291](/Users/boguotong/Library/Application Support/typora-user-images/image-20200629171656291.png)

这时候只需要在Project Structure中Modules中删除掉这个Spring的设置项，重新再从项目的右键点击Add Frameworks Support就能够重新设置了。
