cmd命令生成项目webservice客户端

2020-12-04 java 433 字
命令 webservice

1.webservice接口的检查

能够正常调用的webservice接口是可以直接访问的。在浏览器中输入webservice接口的URL，注意：一定要带上“？wsdl”结尾。调用成功会返回一串xml格式的数据，该数据便是接口具体的方法和返回的类型，如下图所示：

2.cmd命令生成项目webservice客户端

wsimport -s D:\Desktop\temp\s -p com.map -verbose -keep http://ws.webxml.com.cn/WebServices/MobileCodeWS.asmx?wsdl

参数介绍：
-keep：是否生成java源文件
-d：指定.class文件的输出目录
-s：指定.java文件的输出目录
-p：定义生成类的包名，不定义的话有默认包名
-verbose：在控制台显示输出信息
-b：指定jaxws/jaxb绑定文件或额外的schemas
-extension：使用扩展来支持SOAP1.2

注意：
1.指定的文件输出目录一定是要先建好文件夹，不然会报找不到目录；
建议按着项目的路径把这里填上，方便直接复制到项目中。

3.客户端代码的使用

1.执行完2.2的操作后，在指定目录中就会找到生成后的代码，可直接把其拷贝到项目中。
2.在生成的代码中找出接口调用的连接类。一般是WS或Service结尾。打开后会有下图红框的结构。

3.找出其调用类

4.另写一个Service，并在该Service中写一个单例来注入webservice接口的连接类。参考代码如下：

private MobileCodeWSSoap mobileCodeWSSoap;

public MobileCodeWSSoap getMobileCodeWSSoap() {
	        if(mobileCodeWSSoap != null){
	            return mobileCodeWSSoap;
	        }
	        try {
	            URL uurl = new URL("MobileTokenConstants.getValue(MobileTokenConstants.KEY_ADDRESS)");//URL在配置文件中动态配置
	            MobileCodeWS mobileCodeWS = new MobileCodeWS(uurl);
	            mobileCodeWSSoap = mobileCodeWS.getMobileCodeWSSoap();
	        } catch (MalformedURLException e) { 
	            e.printStackTrace();
	        }
	            return mobileCodeWSSoap;
	}
